# EchoDraw · 绘声 — 产品需求文档 (PRD)

> 纯语音控制的 AI 绘图工具。说一句话就能生成场景，并能继续用语音修改。
> 文档版本: v0.1 · 最后更新: 2026-06-13

---

## 1. 产品概述

### 1.1 产品名
**EchoDraw · 绘声**

- 英文名 `EchoDraw`：Echo（回声/语音）+ Draw（绘画）
- 中文名 `绘声`：绘 = 画，声 = 声音，音义双关

### 1.2 一句话定位
**纯语音控制的画布——说一句话就能生成一个场景，并能继续用语音修改它。**

### 1.3 目标用户
- 设计师 / 产品经理做快速概念草图
- 教学场景（老师用语音画示意图）
- 任何想"动嘴不动手"画图的人

### 1.4 核心价值
把"AI 生图"和"绘图工具"打通——AI 负责理解复杂意图，工具负责让人能**继续改**。

差异化点：
- AI 生图工具（Midjourney 等）：能画但改不了
- 传统绘图工具（Figma 等）：能改但要动手
- **EchoDraw：能画 + 能改，全程动嘴**

## 2. MVP 范围

### 2.1 必须有（Must Have）

| 模块 | 能力 |
|---|---|
| 语音输入 | 浏览器原生 Web Speech API，中文优先，按钮开始/结束 |
| 意图理解 | 规则引擎（80% 常见指令）+ LLM 兜底（20% 复杂/模糊指令） |
| 基础绘图 | 圆、矩形、线、椭圆、三角、星形、文本、箭头 |
| 修饰参数 | 颜色、大小、位置、填充、描边 |
| 复合指令 | "画一个红色的大圆在左边" 一次完成 |
| AI 场景生成 | "夜空下的猫咪" → 自动拆解为多个图元组合 |
| 选中与修改 | "刚才那个圆改成蓝色"、"把它移动到中间" |
| 撤销 / 重做 | 语音"撤销"/"重做" |
| 清空 / 保存 | 语音"清空画布"、"保存"（导出 PNG） |

### 2.2 不做（Out of Scope）

- AI 直接生图（B 档，见设计文档说明原因）
- 自由画笔（手绘）
- 图层管理
- 多人协作
- 移动端适配（先桌面 Chrome）
- 复杂动画

## 3. 核心用户场景

### 场景 1：一句话生成场景
> 用户点击麦克风："画一个夜空，有月亮和很多星星"
> 系统：2 秒内画完（深蓝背景 + 月亮 + 30 颗随机星星）

### 场景 2：增量修改
> 用户："把月亮改大一点"
> 用户："月亮改成黄色"
> 用户："再画一只猫坐在月亮下面"

### 场景 3：精确控制
> 用户："画一个半径 100 的圆在 (300, 400)"
> 用户："把它旋转 45 度"

### 场景 4：撤销与回退
> 用户："撤销"
> 用户："重做"

## 4. 命令体系

5 类命令，规则引擎优先处理 1-3 类，LLM 处理 4-5 类。

### Class 1: 创建图元（Create）— 规则引擎
```
画/绘制/加一个/放一个 + [修饰] + 形状
例: 画一个红色的大圆
例: 加一个蓝色的矩形在中间
```

### Class 2: 变换（Transform）— 规则引擎
```
[对象] + 动作 + [参数]
动作: 移动/旋转/缩放/改大小/变色
例: 把它改成蓝色
例: 刚才那个圆放大一倍
```

### Class 3: 选择（Select）— 规则引擎
```
[选择/这个/那个/刚才的/最近的] + [对象]
例: 选中最新的圆
例: 那个红圆
```

### Class 4: 复合场景（AI Scene）— LLM
```
自然语言描述一个场景
例: 画一个温馨的咖啡馆，有桌子、椅子、咖啡杯
```
LLM 输出：结构化的图元列表（JSON），前端逐个绘制。

### Class 5: 模糊/未知 — LLM + 反问
```
画一只猫
→ 系统: 您要画在什么位置？什么颜色？什么大小？
[用户回答后执行]
```

详细命令清单与语法规范见 `docs/DESIGN.md`。

## 5. 技术架构

### 5.1 复用优先清单

| 用途 | 选型 | 理由 |
|---|---|---|
| 画布渲染 | **Konva.js (React-Konva)** | Canvas 库事实标准，节点化便于选中/修改 |
| 语音识别 | **Web Speech API**（浏览器原生） | 零成本，Chrome 支持中文流式 |
| LLM 调用 | **OpenAI-compatible 协议** | 兼容 OpenAI / DeepSeek / Qwen / Moonshot |
| 后端框架 | **Gin** | Go 圈主流，WebSocket 友好 |
| WebSocket | **gorilla/websocket** | 稳定 |
| 前端状态 | **Zustand** | 比 Redux 轻，够用 |
| UI 组件 | **shadcn/ui** | 复制即用，不强依赖 |
| 样式 | **Tailwind** | 快 |
| 构建 | **Vite** | 标准 |
| Schema 校验 | **Zod**（前端）+ **go-playground/validator**（后端） | 校验 LLM 输出 |
| 日志 | **zap**（Go） | 高性能 |

## 7. 风险与缓解

| 风险 | 缓解 |
|---|---|
| Web Speech API 在某些浏览器不可用 | 提示用户切换 Chrome，并提供文本输入框作为降级 |
| LLM 输出结构不稳定 | 用 JSON Schema + Zod 校验，失败时反问 |
| 复杂场景绘制慢 | LLM 规划时限制最多 50 个图元，必要时分批渲染 |
| 语音指令歧义 | 引入"指代消解"（刚才那个/最新创建的），并支持反问 |
| LLM 成本 | 规则引擎优先，LLM 仅兜底；缓存常见场景的拆分结果 |
| 后端无状态部署 | 会话状态放前端 IndexedDB，后端只做指令理解 |

# 附录 A：增量画 / 重画 技术细节（v0.2 补充）

## A.1 概念区分

| 模式 | 含义 | 典型指令 | 实现位置 |
|---|---|---|---|
| **增量画** | 在现有画布上**追加**新图元，已有图元完全保留 | "再画一个圆"、"画一只猫" | LLM `scene` 指令 + 前端 merge |
| **重画** | **清空当前选中对象**，按新指令重画 | "把那个圆改成方形"（=删圆+加方） | 后端拆为 `delete` + `create` |
| **修改** | **就地改属性**，图元 ID 保留 | "把月亮改成黄色" | `update` 指令，仅改 style 字段 |
| **清空** | 整张画布从头来 | "清空画布" | `clear` 指令，前端清空数组 |

**判定规则**（后端规则引擎执行）：
- 指令含 "改/变成/换" + **形状词**（圆/方/三角）→ **重画**（删旧+建新）
- 指令含 "改/变成" + **属性词**（颜色/大小/位置）→ **修改**（就地改）
- 指令含 "再/又/加/放" → **增量**
- 指令含 "重画/重新画" → **清空选中区域 + 增量**

## A.2 画布状态（前端为权威）

```
CanvasState = {
  shapes: Shape[]         // 顺序即 z-order，末尾=最上层
  selectedIds: string[]   // 当前选中
  history: { past: Snapshot[], future: Snapshot[] }  // 撤销/重做栈
}
Snapshot = { shapes: Shape[], selectedIds: string[] }
```

**为什么前端为权威？**
- 撤销/重做必须毫秒级响应，不能 round-trip 后端
- 后端只做"指令理解"，不存画布
- 离线时画布仍能工作（IndexedDB 持久化）

## A.3 撤销 / 重做

- 每次产生**新图元**或**修改图元**或**删除图元**前，push 当前 `Snapshot` 到 `past`
- `undo()`：pop past → current, push current → future
- `redo()`：pop future → current, push current → past
- **重画**（删+建）算两次操作：先压栈再删，再压栈再建，确保可完整回退

## A.4 指代消解（前端执行）

**优先级**（按从高到低匹配）：
1. 显式 ID（前端选中状态）
2. "它 / 这个 / 那个" → 最近的 1 个选中
3. "刚才的 / 最新的 / 最后那个" → shapes 末尾
4. "第一个 / 首个" → shapes 头部
5. "所有 X / 全部 X" → 过滤后的多个
6. 模糊（"那个东西"） → 反问

**过滤器**（可叠加）：
- 颜色（红/蓝/.../hex）
- 形状（圆/方/...）
- 位置（左边/中间/...）
- 创建时间（刚才/之前画的）

## A.5 错误处理与降级

| 失败场景 | 降级方案 |
|---|---|
| 浏览器无 Web Speech API | 显示文本输入框 + 提示切 Chrome |
| Web Speech 识别为空 | 静默忽略，不报错 |
| 规则引擎 + LLM 都失败 | 语音播报"没听清，请再试一次" |
| LLM 返回非法 JSON | Zod 校验，失败则按 `ask_back` 处理 |
| 复杂场景图元数 > 100 | 截断到 100 + 提示"场景过大，已简化" |
| 用户连续反问 ≥ 2 次 | 给出最常见默认值继续执行 |

## A.6 性能预算

| 指标 | 目标 | 实测方法 |
|---|---|---|
| 语音首字延迟 | < 500ms | Chrome DevTools Performance |
| 规则引擎响应 | < 100ms | 后端日志打点 |
| LLM 响应 | < 3s | fetch 计时 |
| 渲染 100 个图元 | < 16ms / frame | Konva layer cache |
| 撤销/重做 | < 50ms | 前端 setState 计时 |

## A.7 增量 vs 重画 判定伪代码

```ts
function classify(text: string, target: Shape[]): "delta" | "redraw" | "modify" | "clear" {
  if (text.match(/清空|清除/)) return "clear"
  if (text.match(/再|又|加|放|新增/)) return "delta"
  if (text.match(/改|变成|换成/) && text.match(/圆|方|三角|星|矩|椭/)) return "redraw"
  if (text.match(/改|变成|设置?为/)) return "modify"
  if (text.match(/重画|重新画/)) return "redraw"
  return "delta"  // 默认增量
}
```
